# OctoAcme Project Management in Copilot Spaces

This repository is focused on scaling institutional knowledge using Copilot Spaces, and the OctoAcme project management process is documented for fast onboarding and consistent execution.

## Quick Overview

- Customer-first strategy
- Iterative delivery (small, incremental releases)
- Clear role-based ownership (PM, PdM, developers, QA, stakeholders)
- Data-informed decisions and continuous improvement
- Psychological safety and proactive communication

## Core Phases

1. Initiation: define problem, goals, stakeholders, and timeline
2. Planning: scope, backlog, risks, milestones, and resource alignment
3. Execution: build, test, review, and iterate with regular syncs
4. Release: deploy, verify, and communicate value
5. Closure: retrospective, documentation updates, and action items

## Core Artifacts

- Project Charter or One-pager
- Roadmap and Release Plan
- Iteration Backlog and Sprint commitments
- Acceptance Criteria and Definition of Done
- Risk Register
- Retrospective notes with action items

## Communication Rhythm

- Weekly PM/PdM check-in
- Team standups (frequency as agreed)
- Monthly stakeholder update
- Ad-hoc escalation channels
